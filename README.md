# Minha-Pagina-Cin
Este repositório é para um **site estático simples** com **informações pessoais e acadêmicas sobre mim** e meu **portfólio**, desenvolvida para ser hospedada na rede do Centro de Informática (CIn) da UFPE. A iniciativa de produzi-la veio da atividade prática extra da disciplina de Desenvolvimento de Software atribuída pelo professor Kiev Gama no dia 09/04/2026 durante a aula, para servir de prática para o projeto que deve ser entregue no final deste 2° semestre.

Dessa forma, era obrigatório que o desenvolvimento desse site fosse com o framework **React**, deixando as outras escolhas de projeto livres, como a estratégia de build e a linguagem de programação utilizada. Para esta primeira ferramenta, o projeto foi criado e compilado com **Vite** no lugar do próprio Create React App pelas seguintes razões: maior velocidade de compilação, uso no mercado cada vez mais padronizado, conforme ele vai substituindo o CRA em diversos cenários e por ser a ferramenta usada no projeto maior da disciplina, o que garante um desenvolvimento mais confortável, graças à adaptação inicial. Já a linguagem de programação escolhida foi **TypeScript** (só poderia ser ela ou o JavaScript) que, além de também estar sendo usada no nosso projeto em equipe, a acho preferível por ser mais organizada e segura devido à rígida tipagem.

---

## Principais dificuldades enfrentadas
Para mim, o maior obstáculo encontrado foi a gestão de tempo para esta atividade. Como também temos, além de outras disciplinas no curso, demandas da vida pessoal, me vi colocando esforço considerável para conseguir finalizar o site com êxito.

Além disso, um bom tempo das etapas iniciais serviu para relembrar o básico de programação web com frameworks e aprender como algumas coisas funcionavam no React, visto que tenho um pouco de experiência com isso por causa do curso técnico.

---

## Principais aprendizados
O primeiro aprendizado está diretamente relacionado com a primeira dificuldade. Pelo tempo de entrega da atividade ser curto, isso reforçou mais ainda o aprendizado de focar em entregar o necessário que foi pedido da forma mais simples possível, o que recorda a filosofia Lean, de certa forma.

Também aprendi bastante sobre o básico em React, pois nunca tinha programado com ele antes (com exceção de React Native, mas não é a mesma coisa). Com isso, também apliquei na prática conceitos essenciais para um desenvolvimento de projeto bem organizado e funcional, especificamento no que diz ao *front-end*, como estrutura de páginas, rotas e estilos com as variáveis CSS.

Definitivamente ainda tenho muito o que aprender e reconheço que há coisas nesse site que devem ser melhoradas, como a mudança de tamanho das imagens no próprio código (o que pode diminuir a performance), classes em CSS que se repetem e disposição de conteúdos no HTML. Porém, para além de fatores técnicos, isso demonstra valorização a uma entrega correspondente às expectativas colocadas, além do fato de que a evolução é sempre constante.

## Uso de IA
Durante o desenvolvimento, houve o uso de duas ferramentas de Inteligência Artificial, mas nenhum código que está aqui foi escrito ou moddificado por uma delas:

1. Copilot do Visual Studio Code: consultado como guia para corrigir pequenos erros que estava com dificuldade de resolver, já que ele tem acesso direto ao projeto.
2. Gemini: esse foi menos usado, tendo sido útil apenas nas etapas iniciais da programação, quando ainda estava testando funcionalidades do React. Nesse caso, tirei uma dúvida sobre o React Router e como ele consegue modificar dinamicamente o estilo dos links de rotas com o seu componente *NavLink*. No final, essa dúvida foi essencial para que eu pudesse definir melhor a estrutura inicial do site.
